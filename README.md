# 🌳 YAKAMI TrustFlow: Smart Contracts para Governança na Amazônia
*(Projeto desenvolvido conforme diretrizes do TrustCode Template Oficial - HackWeb Web3)*

> **Nota de Governança:** Este repositório constitui a versão oficial de submissão do projeto YAKAMI TrustFlow. O histórico completo de desenvolvimento e iterações iniciais pode ser consultado no repositório de trabalho original: [https://github.com/erickthenight/Yakami-TrustFlow-HackWeb](https://github.com/erickthenight/Yakami-TrustFlow-HackWeb).

## 📌 Sobre o desafio & Objetivo
**Objetivo do TrustCode:** Construir Smart Contracts capazes de automatizar acordos, pagamentos e regras de negócio. 
**Foco (YAKAMI):** Na Bacia Amazônica, operações logísticas e manutenções ambientais dependem de validações manuais. Nosso projeto substitui essa confiança informal por um Smart Contract de *Escrow* e Recompensas, utilizando um Gêmeo Digital Imersivo (Unity) como interface de gatilho para funções *on-chain*.

## 🚀 Aplicações da Solução
* **Escrow Corporativo:** Liberação condicional de recursos baseada em validações físicas.
* **Pagamentos Automáticos:** Repasse instantâneo de Tokens YKM para mantenedores locais.
* **Governança e Rastreabilidade:** Registro imutável de compliance logístico e ambiental.

## 🛠 Tecnologias
* **Blockchain:** Solidity, Hardhat, Sepolia Testnet, Remix IDE, OpenZeppelin.
* **Frontend Imersivo:** Unity 3D, Meta XR SDK, C# (Integração Web3).
* **Frontend Web:** Mendix (Plataforma Low-code corporativa).

## 📁 Estrutura do Repositório
* `/contracts` -> Smart Contracts de automação (`RegistroDeUsuariosComRecompensa.sol`) e governança DAO (`YakamiEcosystem.sol`).
* `/scripts` -> Automação e deploy via Hardhat (`deploy.js`).
* `/frontend` -> Integração Web3/Unity (`YakamiMetaverseWeb3.cs`).
* `/assets` e `/docs` -> Dossiê técnico, registros de IP (INPI) e identidades visuais.
* `/test` -> Testes unitários de segurança e lógica de negócio (`RegistroTest.js`).
